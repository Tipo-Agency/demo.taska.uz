# Документация компонентов

## UI Компоненты (components/ui/)

### SafeArea
**Назначение**: Обеспечивает безопасные отступы для системных элементов на мобильных устройствах.

**Зачем отдельно**: 
- Учитывает разные платформы (iOS, Android, Telegram Web App)
- Переиспользуется на всех страницах
- Предотвращает перекрытие контента системными элементами

**Использование**:
```tsx
<SafeArea top bottom>
  <YourContent />
</SafeArea>
```

**Варианты**:
- `SafeAreaTop` - только верхний отступ
- `SafeAreaBottom` - только нижний отступ
- `SafeAreaAll` - все отступы

---

### Container
**Назначение**: Базовый контейнер для контента с максимальной шириной и центрированием.

**Зачем отдельно**:
- Единообразные отступы на всех страницах
- Адаптивная ширина
- Опциональная поддержка safe areas

**Использование**:
```tsx
<Container maxWidth="xl" padding safeArea>
  <YourContent />
</Container>
```

---

### PageLayout
**Назначение**: Базовый layout для страниц с header, content, footer.

**Зачем отдельно**:
- Единая структура всех страниц
- Автоматические safe areas
- Правильное распределение пространства

**Использование**:
```tsx
<PageLayout
  header={<Header />}
  footer={<Footer />}
>
  <YourContent />
</PageLayout>
```

---

### Button
**Назначение**: Универсальная кнопка со всеми вариантами стилей.

**Зачем отдельно**:
- Единый стиль кнопок во всем приложении
- Поддержка всех вариантов (primary, secondary, danger, ghost, outline)
- Иконки, размеры, состояния

**Использование**:
```tsx
<Button variant="primary" size="md" icon={Plus} onClick={handleClick}>
  Создать
</Button>
```

---

### Input
**Назначение**: Поле ввода с поддержкой label, error, иконок.

**Зачем отдельно**:
- Единый стиль полей ввода
- Валидация и отображение ошибок
- Поддержка иконок

**Использование**:
```tsx
<Input
  label="Email"
  type="email"
  error={errors.email}
  icon={Mail}
  value={email}
  onChange={(e) => setEmail(e.target.value)}
/>
```

---

### Select
**Назначение**: Выпадающий список для выбора опций.

**Зачем отдельно**:
- Единый стиль с Input
- Поддержка label и error
- Адаптивный дизайн

**Использование**:
```tsx
<Select
  label="Статус"
  options={statusOptions}
  value={status}
  onChange={(e) => setStatus(e.target.value)}
/>
```

---

### Toast
**Назначение**: Временные уведомления для пользователя.

**Зачем отдельно**:
- Централизованная система уведомлений
- Разные типы (success, error, info, warning)
- Автоматическое скрытие

**Использование**:
```tsx
<Toast
  message="Задача создана"
  type="success"
  duration={3000}
  onClose={() => setShowToast(false)}
/>
```

---

### StandardModal
**Назначение**: Модальное окно для всех случаев использования.

**Зачем отдельно**:
- Единый стиль модальных окон
- Поддержка safe areas на мобильных
- Breadcrumbs для навигации
- Адаптивные размеры

**Использование**:
```tsx
<StandardModal
  isOpen={isOpen}
  onClose={handleClose}
  title="Создать задачу"
  size="md"
  footer={<Button onClick={handleSave}>Сохранить</Button>}
>
  <YourForm />
</StandardModal>
```

---

## Принципы использования

1. **Всегда используй UI компоненты** вместо прямого написания HTML
2. **Не дублируй стили** - если что-то повторяется, создай компонент
3. **Композиция над наследованием** - собирай сложное из простого
4. **Мобильная версия в приоритете** - используй SafeArea везде
